Use other JavaScript libraries

When a third-party library is used, If it is shared by many widgets, place the library under the stemapp/libs folder of the app. If it is an plain JavaScript file, put the library link into the resources array in the init.js file.

resources = resources.concat([
  {type: 'css', url: apiUrl + 'js/dojo/dojo/resources/dojo.css'},
  {type: 'css', url: apiUrl + 'js/dojo/dijit/themes/claro/claro.css'},
  {type: 'css', url: apiUrl + 'js/esri/css/esri.css'},
  {type: 'js', url: apiUrl},
  {type: 'css', url: path + 'jimu.js/css/jimu.css'}
]);

If the library is an AMD package, you can configure the package at dojoConfig in the init.js.

dojoConfig = {
    parseOnLoad: false,
    async: true,
    tlmSiblingOfDojo: false,

    has: {
      'extend-esri': 1
    },
    packages : [{
      name : "widgets",
      location : path + "widgets"
    }, {
      name : "jimu",
      location : path + "jimu.js"
    }, {
      name : "themes",
      location : path + "themes"
    }]
  }

If the library is used by one widget only, you can put the library in the widget's folder and add it in the widget's define dependencies array.
